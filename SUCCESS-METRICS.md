# Success Metrics: GitHub Training Curriculum for Farmers

## Overview

This document defines the measurement framework for evaluating the GitHub training curriculum using the Kirkpatrick 4-level evaluation model, adult learning assessment methods, and accessibility compliance tracking. Metrics span learner reactions, skill acquisition, behavioral change, and farm productivity impact over 3-6 months.

**Evaluation Framework:** Kirkpatrick Model (Reaction, Learning, Behavior, Results)

**Measurement Philosophy:** Multi-level assessment tracking progression from immediate satisfaction to long-term farm operational improvements, with accessibility and inclusivity as foundational success criteria.

## Primary Metrics

### 1. Learner Completion Rate (Kirkpatrick Level 1)

| Attribute | Value |
|-----------|-------|
| **Definition** | Percentage of enrolled farmers who complete 8 or more training sessions |
| **Unit** | Percentage |
| **Baseline** | 0% (no training delivered) |
| **Target** | 75% completion rate |
| **Improvement** | +75 percentage points |
| **Measurement Method** | Session attendance tracking; learner considered "complete" if attended 8+ of 12 sessions |
| **Data Source** | Session attendance sheets, sign-in records |
| **Frequency** | Weekly during delivery phase (Months 4-6) |
| **Formula** | (Number of learners attending 8+ sessions / Total enrolled learners) × 100 |
| **Dashboard Widget** | Progress bar + numeric percentage |
| **Owner** | Curriculum Developer/Instructor |

**Industry Benchmark:** 45-60% average completion for technical training; 75-85% top quartile for experiential, hands-on training (Domain Research)

**Success Threshold:** 75% completion rate indicates effective engagement strategies, appropriate pacing, and successful seasonal scheduling

---

### 2. Session Satisfaction Score - Net Promoter Score (NPS) (Kirkpatrick Level 1)

| Attribute | Value |
|-----------|-------|
| **Definition** | Learner likelihood to recommend training to peer farmers, measured via NPS scale |
| **Unit** | Net Promoter Score (-100 to +100) |
| **Baseline** | N/A (no prior training) |
| **Target** | NPS +60 or higher |
| **Improvement** | Establish positive recommendation rate |
| **Measurement Method** | Post-session survey question: "How likely are you to recommend this training to another farmer?" (0-10 scale) |
| **Data Source** | Session feedback surveys (paper or digital) |
| **Frequency** | After each session (8-12 times over delivery phase) |
| **Formula** | % Promoters (9-10 rating) - % Detractors (0-6 rating) |
| **Dashboard Widget** | Gauge chart with color zones (red <0, yellow 0-49, green 50+) |
| **Owner** | Curriculum Developer/Instructor |

**Industry Benchmark:** NPS +30-50 is typical for training programs; +60+ indicates exceptional learner satisfaction

**Success Threshold:** NPS +60 demonstrates strong learner engagement, relevance to farming operations, and effective adult learning principles application

### 7. Cognitive Load Assessment (Learner Confidence)

| Attribute | Value |
|-----------|-------|
| **Definition** | Learner self-reported confidence in GitHub skills, measured via post-session survey |
| **Unit** | Likert scale average (1-5, where 5 = very confident) |
| **Baseline** | 1.0 (no confidence pre-training) |
| **Target** | 4.0+ average confidence |
| **Improvement** | +3.0 points |
| **Measurement Method** | Post-session survey question: "How confident do you feel using the GitHub skills taught today?" (1-5 scale) |
| **Data Source** | Session feedback surveys |
| **Frequency** | After each session (8-12 measurements) |
| **Formula** | Sum of all confidence ratings / Number of responses |
| **Dashboard Widget** | Line chart showing confidence progression across sessions |
| **Owner** | Curriculum Developer/Instructor |

**Cognitive Load Theory Application:** Confidence scores indicate whether content chunking, pacing, and scaffolding are successfully managing learner mental effort. Declining confidence signals cognitive overload requiring immediate intervention.

**Success Threshold:** 4.0+ average confidence demonstrates effective application of CLT principles, appropriate pacing, and successful anxiety management

## Appendix: Measurement Instruments

### A. Post-Session Feedback Survey (Kirkpatrick Level 1)

**Format:** 5-minute paper or digital survey
**Frequency:** After each session

**Questions:**
1. **Satisfaction:** "How valuable was today's session for your farm operations?" (1-5 scale)
2. **Relevance:** "Were the examples and exercises relevant to your farming work?" (Yes/Somewhat/No)
3. **Pacing:** "Was the session pacing appropriate?" (Too fast / Just right / Too slow)
4. **Confidence:** "How confident do you feel using the GitHub skills taught today?" (1-5 scale)
5. **Net Promoter Score:** "How likely are you to recommend this training to another farmer?" (0-10 scale)
6. **Open Feedback:** "What worked well today? What should be improved?"

---

### B. Skill Demonstration Checklist (Kirkpatrick Level 2)

**Format:** Instructor observation checklist during hands-on practice
**Frequency:** Every session (formative assessment)

**Session 1-2 Example (Repository Navigation & Issues):**
- [ ] Learner navigates to a GitHub repository without assistance
- [ ] Learner locates and opens a file in the repository
- [ ] Learner creates a new issue with title and description
- [ ] Learner assigns issue to themselves
- [ ] Learner adds a label to the issue

**Pass Criteria:** Learner completes 4 of 5 tasks independently (80%)

---

### C. 30-Day Skill Retention Assessment (Kirkpatrick Level 2)

**Format:** Brief practical demonstration (10-15 min per learner, video call or in-person)
**Frequency:** One-time at 30 days post-final session

**Tasks:**
1. "Show me how to create a GitHub issue for a farm task"
2. "Add that issue to a project board"
3. "Assign the issue to yourself"
4. "Show me where you would find notifications about updates"

**Pass Criteria:** Learner completes 3 of 4 tasks independently (75%)

---

### D. 60-Day GitHub Usage Survey (Kirkpatrick Level 3)

**Format:** 10-minute survey (paper or digital)
**Frequency:** One-time at 60 days post-final session

**Questions:**
1. "Are you currently using GitHub for farm operations?" (Yes/No)
